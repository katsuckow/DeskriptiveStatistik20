Beispiellösung

# Datei in die Variable vp einlesen
vp = read.table("vp.dat", header = T)

# kurzer Überblick über die Daten 
head(vp)

#Aufgabe: Mittelwerte der beiden Bedingungen (cond) auf Interest Area (pos) 6 anzeigen lassen
# verschiedene Bedingungen anschauen
table(vp$cond)

# unterschiedliche Interest Areas anschauen
table(vp$pos)

# Mittelwert der Lesezeiten auf Interest Area 6 für die Bedingung a
vp[vp$pos == 6,]
vp[(vp$pos == 6) & (vp$cond == "a"), ]
mean(vp[(vp$pos == 6) & (vp$cond =="a"),8])

# Mittelwert der Lesezeiten auf Interest Area 6 für die Bedingung b
vp[(vp$pos == 6) & (vp$cond =="b"),]
vp[(vp$pos == 6) & (vp$cond == "b"),8]
mean(vp[(vp$pos == 6) & (vp$cond == "b"),8])
# folgender Befehl nicht Teil der Aufgabe. Was kann man noch weiter darstellen
vp = read.table("vp.dat", header = T)
barplot(c(mean(vp[(vp$pos == 6) & (vp$cond =="a"),8]), mean(vp[(vp$pos == 6) & (vp$cond == "b"),8])), names.arg = c("a","b"))

9.2 Datenvisualisierungen mit ggplot()

Grafiken
Graphiken können bei der Auswertung erhobener Daten hilfreich sein:

  • für die Datenexploration (eye-balling der Daten)
  • sie erleichtern das Auffinden von Strukturen in den Daten
  • zum Prüfen von Voraussetzung für Verfahren der Inferenzstatistik
  • zur leicht verständlichen Kommunikation der Ergebnisse

Es gibt viele unterschiedliche Möglichkeiten bei der Datenvisualisierung, zB.:

  • die Zuordnung von “ästhetischen Eigenschaften”" (z.B. Farben, Formen, Größen) zu den Daten
  • geometrische Objekte (Punkte, Linien, Balken)
  • statistische Transformationen der Daten (z.B. Darstellung von Mittelwerten, Standardabweichungen)
  • der Darstellung getrennt nach verschiedenen Gruppen/Facetten (verschiedene Untergruppen, welche in den Daten vorhanden sind)
  • einem Koordinatensystem

9.3 Grafiken mit ggplot()

Datensatz sus der Hausaufgabe 

library(ggplot2)
rt = read.table("vp.dat",header = T)
rt5 = rt[rt$pos == 5,]
ggplot(data = rt5) + geom_point(mapping = aes(x = cond, y = time, color = cond))

Viele Datenpunkte werden übereinander dargestellt. Dadurch kann man die Verteilung der Datenpunkte kaum einschätzen. Dieses Overplotting lässt sich reduzieren.

rt = read.table("vp.dat",header = T)
rt5 = rt[rt$pos == 5,]
ggplot(data = rt5) + geom_jitter(mapping = aes(x = cond, y = time, color = cond))

Die Breite von geom_jitter() lässt sich auch festlegen.

library(ggplot2)
rt = read.table("vp.dat",header = T)
rt5 = rt[rt$pos == 5,]
ggplot(data = rt5) + geom_jitter(mapping = aes(x = cond, y = time, color = cond), width = 0.1)

Man kann den Unterschied zwischen den beiden Bedingungen auch für unterschiedliche Gruppen darstellen. Das kann, wie Ihr seht, auch schnell unübersichtlich werden. Im folgenden Code wird die Darstellung zwischen den beiden Bedingungen separat für jede Versuchsperson dargestellt.

rt = read.table("vp.dat",header = T)
rt5 = rt[rt$pos == 5,]
ggplot(data = rt5) + geom_jitter(mapping = aes(x = cond, y = time, color = cond), width = 0.1) + facet_wrap( ~ vp)

Eine solche Darstellung, obwohl auf den 1. Blick unübersichtlich, kann hilfreich sein, um den Effekt bei jeder Versuchspersonen einzuschätzen.

Eine solche Unterteilung kann natürlich auch für die getesten items hilfreich sein. Besonders bei der Untersuchung der Frage, ob der Effekt bei einzelnen experimentellen Items von den anderen abweicht.

library(ggplot2)
rt = read.table("vp.dat",header = T)
rt5 = rt[rt$pos == 5,]
ggplot(data = rt5) + geom_jitter(mapping = aes(x = cond, y = time, color = cond), width = 0.1) + facet_wrap( ~ item)

stat_summary()
Mit der stat_summary() Funktion lassen sich weitere Informationen (z.B. Mittelwerte, Standardabweichungen) zum besseren Vergleich zwischen Bedingungen einem Barplot zufügen.

library(ggplot2)
library(Hmisc)
rt = read.table("vp.dat",header = T)
rt5 = rt[rt$pos == 5,]
ggplot(data = rt5, aes(cond,time,fill = cond)) + stat_summary(fun = mean, geom = "bar")

Zu dem obigen Beispiel lassen sich auch die Standardabweichungen dazuplotten, um den Unterschied zwischen Mittelwerten besser einschätzen zu können. Dazu kann man auch die stat_summary Funktion benutzen und dann auch die Errorbars darstellen.

rt = read.table("vp.dat",header = T)
rt5 = rt[rt$pos == 5,]
ggplot(data = rt5, aes(cond,time,fill = cond)) + stat_summary(fun = mean, geom = "bar") + stat_summary(fun.data = mean_cl_normal, geom = "errorbar",width = 0.2)

Nachdem nun viele unterschiedliche Grafiken in R erstellt wurden, soll der folgende Code zeigen, wie man eine Grafik im aktuellen Verzeichnis des Workspaces exportieren und speichern kann.



Die pdf() Funktion erstellt ein .pdf Dokument. Wenn eine plot Funktion aufgerufen wird (im unteren Beispiel ggplot()), wird das Ergebnis in eine .pdf Datei geschrieben. Mit dem dev.off() wird das Zeichnen der Grafik abgeschlossen. Bei dieser Form eine Grafik zu speichern oder exportieren, ist der dev.off() Befehl extrem wichtig, sonst wird die Grafik nicht erstellt.

# dieser Code speichert die erstellte Grafik als pdf in Euren aktuellen Workspace
rt = read.table("vp.dat",header = T)
rt5 = rt[rt$pos == 5,]
pdf("barplot_pos5.pdf")
ggplot(data = rt5, aes(cond,time,fill = cond), + stat_summary(fun = mean, geom = "bar") + stat_summary(fun.data = mean_cl_normal, geom = "errorbar",width = 0.2))
dev.off()

Der folgende Code zeigt den Lesezeitunterschied zwischen den Bedingungen an Position 5 für alle unterschiedlichen Items an.

rt = read.table("vp.dat",header = T)
rt5 = rt[rt$pos == 5,]
ggplot(data = rt5, aes(cond,time,fill = cond)) + stat_summary(fun = mean, geom = "bar") + facet_wrap( ~ item)

9.4 ggplot() Übersicht über Parameter und Funktionen

Zuordnung zu Variable Zuweisung von spezifischem Wert
x-Achse x = Variable Werte, die auf X-Achse abgebildet werden
y-Achse y = Variable Werte, die auf Y-Achse abgebildet werden
colour colour = variable colour = “name”(z.B.“red”, “blue”)
alternativ als hexadezimale Farbdefinition:
colour = “\(\#\)RRGGBB”,“\(\#\)FF00FF”
shape shape = variable shape = Wert (Ganze Zahl, z.B. 3 für +)
shape = “x”
(z.B. shape = “T” bewirkt, dass
Datenpunkte als “T” dargestellt werden)
size size = variable size = Wert (Größe in mm)
linetype linetype = variable linetype = Wert (Ganze Zahl,
z.B. 2 für gestrichelte Linie oder “dotted”)

geom notwendige Spezifikationen optionale Spezifikation

geom_line x: Variable auf der x-Achse;
y: Variable auf der y-Achse 		colour, size, linetype, alpha \(\dots\)

geom_histogram x: Variable auf der x-Achse colour, size, fill, linetype, alpha \(\dots\)
geom_boxplot x: Variable auf der x-Achse colour, size, fill, alpha \(\dots\)
geom_text x: horizontale Koordinate; y: vertikale Koordinate; label: Text, der dargestellt werden soll colour, size, angel, hjust, vjust, alpha \(\dots\)

9.7 stat-Spezifikationen II

stat Spezifikationen optionale Spezifikationen
stat_summary fun: legt Funktion fest, deren Ergebnis auf y-Achse abgetragen wird (z.B. mean, median, \(\dots\)) shape, colour, size, alpha, \(\dots\)
stat_identity Gibt einzelne Rohwerte zurück default bei geom_point()
stat_bin Gibt aus:
- count: Anzahl der Datenpunkte innerhalb eines bins
- density: Häufigkeitsdichte
Nützliche Spezifikationen: binwidth (Breite der bins), bins (Anzahl der bins)
stat_boxplot Gibt aus: ymin (unterer Whisker); lower (unterer Angelpunkt hinge), middle (Median), upper (oberer Angelpunkt hinge), ymax (oberer Whisker) default bei geom_boxplot
